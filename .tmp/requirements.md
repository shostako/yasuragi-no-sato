# やすらぎの郷 - 要件定義書

## 1. プロジェクト概要

### 1.1 サイト情報
- **施設名**: やすらぎの郷（YASURAGI NO SATO）
- **サイト種別**: 介護施設コーポレートサイト
- **目的**:
  - 施設の認知度向上
  - サービス内容の訴求
  - 見学予約・問い合わせの獲得
  - 採用活動の支援

### 1.2 ターゲットユーザー
| ユーザー層 | ニーズ |
|-----------|--------|
| 高齢者の家族 | 親の介護サービスを探している |
| ケアマネージャー | 利用者に紹介する施設を探している |
| 高齢者本人 | 自身のケアサービスを検討している |
| 求職者 | 介護職の求人を探している |

### 1.3 技術スタック
- **フレームワーク**: Next.js 16 (App Router)
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS 4
- **バックエンド/DB**: Supabase (PostgreSQL)
- **認証**: Supabase Auth
- **デプロイ**: Vercel

## 3. 機能要件

### 3.1 基本機能

#### お問い合わせフォーム
- **入力項目**:
  - お名前（必須）
  - フリガナ（必須）
  - 電話番号（必須）
  - メールアドレス（必須）
  - お問い合わせ種別（選択: 見学予約/サービス相談/採用/その他）
  - お問い合わせ内容（必須）
  - 個人情報同意チェック（必須）
- **送信先**: Supabase DB + 管理者メール通知
- **確認画面**: あり
- **完了画面**: あり

#### お知らせ機能
- **記事属性**:
  - タイトル
  - カテゴリ（お知らせ/イベント/採用/その他）
  - 公開日
  - 本文（リッチテキスト）
  - サムネイル画像（任意）
- **一覧表示**: 最新順、ページネーション（10件/ページ）
- **管理**: Supabase管理画面または専用CMS

### 3.2 将来機能（Phase 2）

#### 見学予約システム
- カレンダーUIでの日時選択
- 予約枠の管理（管理画面）
- 予約確認メール自動送信
- リマインダーメール

#### 会員ログイン
- 家族向けポータル
- 利用者情報の閲覧
- 請求書・領収書のダウンロード

## 4. 非機能要件

### 4.1 パフォーマンス
- **LCP**: 2.5秒以内
- **FID**: 100ms以内
- **CLS**: 0.1以内
- **画像最適化**: Next.js Image使用、WebP対応

### 4.2 SEO
- **メタデータ**: 全ページに適切なtitle/description設定
- **構造化データ**: LocalBusiness、FAQPage
- **サイトマップ**: 自動生成
- **robots.txt**: 設定

### 4.3 アクセシビリティ
- **WCAG 2.1 AA準拠**を目標
- セマンティックHTML
- 適切なコントラスト比
- キーボードナビゲーション対応
- スクリーンリーダー対応

### 4.4 セキュリティ
- HTTPS必須
- フォームのCSRF対策
- 入力値バリデーション
- SQLインジェクション対策（Supabase RLS）

### 4.5 レスポンシブ対応
- **ブレークポイント**:
  - Mobile: < 640px
  - Tablet: 640px - 1024px
  - Desktop: > 1024px
